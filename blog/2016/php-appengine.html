Twig

On AppEngine a PHP file cannot write to the filesystem because of security and scalability. An this is good practice, look at all PHP code injection that causes a lot of trouble in other PHP based systems. But Twig needs a cache to precompile the template files into PHP code that can be executed.

The solution is to precompile all the templates before going to production. Every template get a unique classname based on the path of the file and the filename. But as I noticed this unique code differs on the development environment and production environment. After some digging in the Twig I found the reason behind this.

In the file Twig/Loader/Filesystem.php:198 in the function doFindTemplate is the following code:


if (false !== $realpath = realpath($path.'/'.$shortname)) {
 $this->cache[$name] = $realpath;
 return true;
}

This code searches trough the paths and tries to find the template. The problem here is the call to the function realpath. This converts a relative path to the absolute path. And this path is used to base the filename on.

When those lines are commented out the pathname stays relative and the calculated unique code is the same on production and development.

But you must precompile the template files before they can be executed. Therefore you should create a script to do that.

require_once('vendor/autoload.php');

$loader = new Twig_Loader_Filesystem('templates');

$twig = new Twig_Environment($loader, array(
  'cache' => 'templates_c',
));

$twig->loadTemplate('index.html.twig');
