Date Time handling in Java

Posted on 08 April 2016

The first thing I learned from working with dates is that it is really hard.
In the beginning I would resort in calculations like 86400 * $days and calculate a new date by hand. Maybe it sounds familiar but I soon discovered that it introduces more problems than it solves. Calculating a leap year for example is difficult.

Since JVM 1.8 there is an easy API that supports calculating with dates and times. When you don't need timezones you can use the LocalDate, LocalTime

LocalDate

LocalDate date = LocalDate.now();
LocalDate newDate = date.plusYears(10);

LocalDate is an immutable object, this means that an instance of LocalDate cannot be changed. Every plus/minus method returns a new instance.
The major advantage is that every instance is now thread safe without any difficult locking.

LocalTime

Working with the times works in the same way as the dates. 

LocalTime lt = LocalTime.now();
lt.isBefore(LocalTime.NOON);

LocalTime lt = LocalTime.now();
LocalTime lt2 = lt.plusHours(12);
